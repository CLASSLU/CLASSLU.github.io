# 武汉久玩电商官网设计规格（单页）

- 日期：2026-03-28
- 项目：武汉久玩电子商务有限公司官网
- 形态：单页官网（中英双语）
- 核心目标：品牌背书展示（提升信任与专业形象）
- 风格方向：温和服务风（亲和、专业、可信）

## 1. 目标与范围

### 1.1 业务目标
1. 清晰传达公司定位与服务能力。
2. 建立“专业、稳定、可长期合作”的品牌认知。
3. 提供明确联系入口，降低商务沟通门槛。

### 1.2 本期范围（In Scope）
1. 单页结构官网（Desktop + Mobile）。
2. 中英双语即时切换（默认中文）。
3. 展示固定内容模块：公司介绍、能力、流程、背书、联系方式。
4. 轻量动画与高可读性排版。

### 1.3 非范围（Out of Scope）
1. CMS 后台管理。
2. 在线表单提交与后端接口。
3. 多页面站点与复杂 SEO 策略。

## 2. 用户与使用场景

### 2.1 目标访客
1. 潜在品牌合作方（主要）。
2. 行业合作伙伴。
3. 对企业资质与能力进行初筛的商务/采购角色。

### 2.2 关键场景
1. 首次访问，快速判断公司是否可靠。
2. 了解服务覆盖和执行方法。
3. 获取联系方式，进入下一步合作沟通。

## 3. 信息架构（单页）

1. Header（品牌标识、锚点导航、语言切换）
2. Hero（品牌主张、公司定位、CTA）
3. About（我们是谁）
4. Capabilities（核心能力）
5. Process（合作流程）
6. Trust（品牌背书与服务理念）
7. Contact（联系我们）
8. Footer（版权与补充说明）

## 4. 视觉与品牌表达

### 4.1 色彩
1. 主色：`#2C8E83`（暖青绿）
2. 辅色：`#F6F3ED`（米白）、`#1F2A2E`（深墨）、`#F2A65A`（柔和橙）
3. 背景：浅色渐变 + 柔和装饰形状

### 4.2 字体
1. 中文：Noto Sans SC（正文字体）
2. 中文标题点缀：ZCOOL XiaoWei
3. 英文：Manrope（正文）
4. 英文标题点缀：Cormorant Garamond

### 4.3 气质与节奏
1. 专业可信 + 服务温度。
2. 通过留白、圆角卡片、柔和阴影降低压迫感。
3. 避免强科技冷感与夸张视觉噪音。

## 5. 双语文案策略

### 5.1 默认与切换
1. 默认语言：中文（`zh`）
2. 次语言：英文（`en`）
3. 用户选择写入 `localStorage`，下次访问自动恢复。

### 5.2 核心文案（摘要）
1. Hero 中文：让品牌在电商平台被看见、被信任、被选择。
2. Hero 英文：Help brands be seen, trusted, and chosen in e-commerce.
3. 联系方式：
   - 邮箱：`yuanluge@2925.com`
   - 地址：`武汉市江夏经济开发区南湖路锦绣良缘B4幢1层1-101`

## 6. 交互设计

1. 顶部导航吸顶，滚动后增加轻微毛玻璃效果。
2. 模块进入视口后分段淡入（stagger reveal）。
3. 语言切换不跳页，文本即时替换。
4. CTA 在首屏与页尾双入口可见。
5. 遵循 `prefers-reduced-motion`，减少动态效果。

## 7. 技术方案

### 7.1 技术栈
1. HTML5 + CSS3 + 原生 JavaScript。
2. 文件结构：`index.html`、`styles.css`、`script.js`。

### 7.2 组件划分
1. Header/Nav
2. Hero
3. About
4. Capabilities（4 卡片）
5. Process（4 步）
6. Trust
7. Contact
8. Footer

### 7.3 i18n 实现
1. 在 JS 中维护 `translations = { zh: {...}, en: {...} }`。
2. 页面元素使用 `data-i18n` 声明键名。
3. 切换时遍历 `data-i18n` 节点并替换文案。

### 7.4 响应式
1. 断点：1200 / 992 / 768 / 480。
2. 移动端导航折叠为汉堡菜单。
3. 卡片网格自动降列，保证触控可用性。

## 8. 数据流与状态

1. 初始加载：读取 `localStorage.language`，默认 `zh`。
2. 渲染阶段：按当前语言映射全部文本。
3. 用户切换语言：
   - 更新内部状态 `currentLang`
   - 重新渲染文本
   - 写回 `localStorage`
4. 其他交互（导航、动画）均为前端本地状态，无后端依赖。

## 9. 异常与容错

1. `localStorage` 不可用时：退回默认中文。
2. 某个翻译键缺失时：回退到中文文本。
3. 自定义字体加载失败时：回退系统字体栈。
4. JS 不可用时：页面仍以中文静态内容可读。

## 10. 测试与验收

### 10.1 功能验收
1. 所有模块完整渲染。

3. 联系方式展示正确且可复制。
4. 导航锚点可达且滚动平滑。

### 10.2 兼容与体验
1. Desktop（>=1200）、Tablet（~768）、Mobile（<=480）布局正常。
2. 无明显重叠、截断、错位。
3. 动画不影响可读性与性能。

### 10.3 基线质量
1. 可访问性：语义结构清晰，按钮可聚焦。
2. 性能：静态资源轻量，首屏加载快速。
3. 最佳实践：无阻塞脚本错误。
